# REQUIREMENTS — Feature 3: Candidate Profile Breakdown

## EARS Format

### REQ-019

> When the recommendation queue is displayed, the system shall show each candidate's name, primary role, core skills, availability label, current allocation percentage, and total suitability score.

### REQ-020

> When a user clicks the "View Breakdown" or "Expand Breakdown" toggle on a candidate card, the system shall expand an accordion displaying the raw values calculated for `S_Skill`, `S_Avail`, `S_Role`, and `S_Total`.

### REQ-021

> While the breakdown panel is open, the system shall display a short explanation of why the candidate was recommended.

### REQ-022

> Where a candidate's current allocation percentage is greater than 50%, the system shall show the availability constraint clearly on the candidate card.

### REQ-023

> The system shall display score bars or labels for the scoring factors used in the suitability calculation.
