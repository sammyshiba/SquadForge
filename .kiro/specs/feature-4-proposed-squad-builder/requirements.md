# REQUIREMENTS — Feature 4: Proposed Squad Builder

## EARS Format

### REQ-024

> When a user clicks the "Assign to Squad" button on a candidate card, the system shall append that candidate profile to the Proposed Delivery Squad array.

### REQ-025

> If a candidate has already been added to the Proposed Delivery Squad array, then the system shall prevent duplicate assignment and show the candidate as already assigned.

### REQ-026

> While a candidate is appended to the Proposed Delivery Squad array, the system shall render their profile inside the sticky footer tray workspace.

### REQ-027

> When a user clicks the "Remove" action inside the sticky footer tray on an assigned candidate, the system shall remove that candidate from the Proposed Delivery Squad array.

### REQ-028

> While candidates are assigned to the Proposed Delivery Squad, the system shall display the number of filled squad seats.

### REQ-029

> When the Proposed Delivery Squad contains at least one candidate, the system shall enable the Finalize Squad and Export Spec actions.

### REQ-030

> When the user finalizes the squad, the system shall show a confirmation summary of the selected candidates and their suitability scores.

### REQ-031

> When the user resets the proposal, the system shall clear the Proposed Delivery Squad array and return the sticky footer tray to an empty state.
